* Components:
  • Authentication Service (Handles login requests and validates credentials)
  • Session Store (Manages user sessions and persists them)
  • User Repository (Stores user data)
  • API Gateway (Serves as the entry point for user requests)

* Workflow:
  1. User requests login through API Gateway
  2. API Gateway routes request to Authentication Service
  3. Authentication Service verifies credentials with User Repository
  4. Authentication Service returns login response to API Gateway
  5. API Gateway returns response to user's browser
  6. If credentials are valid, Authentication Service creates and stores a new session in Session Store
  7. If credentials are invalid, Authentication Service returns an error response

* Tech Stack:
  • Frontend: HTML, CSS, JavaScript (with libraries like React or Angular)
  • Backend: Node.js with Express.js (for API Gateway and Authentication Service)
  • Database: Relational database like MySQL or PostgreSQL (for User Repository)
  • Session Store: In-memory data store like Redis
