### System Design

* **Components:**
  • Frontend (Client-side): Responsible for handling user input, validating credentials, and communicating with the backend.
  • Backend (Server-side): Handles user authentication, session management, and database interactions.
  • Database: Stores user credentials and session information.
  • API (Application Programming Interface): Exposes endpoints for authentication and session management.

* **Workflow:**
  1. User submits login credentials (email and password) through the frontend.
  2. Frontend sends the credentials to the backend via API.
  3. Backend authenticates the credentials and creates a new session if valid.
  4. Backend stores the session information in the database.
  5. Backend returns a session token to the frontend.
  6. Frontend stores the session token and uses it to access protected routes.

* **Tech Stack:**
  • Frontend: React, JavaScript, CSS
  • Backend: Node.js, Express.js, MongoDB
  • Database: MongoDB
  • API: RESTful API with JSON Web Tokens (JWT)
